# PLECS–Octave Automated Parameter Sweep Framework

## Overview

This framework provides a **Python-driven automation pipeline** that generates Octave scripts and injects them directly into **PLECS simulation models**. The goal is to perform **automated parameter sweep simulations** without manually modifying the PLECS model for each parameter set.

The tool enables:

* Automated **parameter sweep simulations**
* Extraction of **model variable mappings**
* Automatic generation of **Octave simulation scripts**
* Direct **PLECS script injection**
* Structured storage of simulation results

Typical applications include:

* Design validation
* Sensitivity analysis
* Worst-case analysis (WCA)
* Parameter optimization
* Regression testing

---

# System Architecture

The framework consists of several layers responsible for data preparation, script generation, and simulation execution.

```mermaid
flowchart LR

A[Python Controller] --> B[Mapping Extraction]
B --> C[Struct Generator]
C --> D[Octave Script Template Engine]
D --> E[Generated Octave Script]

E --> F[PLECS Script Injector]
F --> G[PLECS Model]

G --> H[Octave Simulation Execution]
H --> I[Simulation Results]
```

### Architecture Components

| Component         | Description                                     |
| ----------------- | ----------------------------------------------- |
| Python Controller | Orchestrates the complete workflow              |
| Mapping Extractor | Parses parameter definitions from ScriptBody.py |
| Struct Generator  | Creates hierarchical Octave structures          |
| Template Engine   | Generates Octave scripts using Jinja2           |
| Script Injector   | Injects generated scripts into PLECS models     |
| PLECS Engine      | Executes simulations                            |
| Results Storage   | Stores simulation outputs                       |

# Core Modules

## inject_octave_simple()

### Purpose

Injects a generated Octave script into a `.plecs` model file.

### Function Signature

```python
inject_octave_simple(
    plecs_file_path,
    output_file_path,
    m_file_path
)
```

### Injection Logic

```mermaid
flowchart TD

A[Read PLECS File] --> B{Script Section Exists?}

B -->|Empty Script| C[Replace Script Content]
B -->|Existing Script| D[Append New Script Section]
B -->|No Script Section| E[Insert Before Final Brace]

C --> F[Write Updated File]
D --> F
E --> F
```

# Parameter Mapping Parser

The mapping parser extracts parameter assignments from `ScriptBody.py`.

### Required Marker Section

```
#! -------------------------------------------------------------------------------------Don't change above this line---------------------------------------------------------------------------

mdlVars["Common"]["Water_Temperature"] = mapVars[X1][0]

#! -------------------------------------------------------------------------------------Don't change under this line---------------------------------------------------------------------------
```

### Parser Logic

```mermaid
flowchart TD

A[Open ScriptBody.py] --> B[Locate Marker Section]
B --> C[Split Section Into Lines]

C --> D{Contains Assignment?}

D -->|No| C
D -->|Yes| E[Extract mdlVars Path]

E --> F{Derived Parameter?}

F -->|Yes| G[Convert mdlVars References]
F -->|No| H[Convert mapVars References]

G --> I[Generate Octave Expression]
H --> I

I --> J[Store Mapping]
```

# Parameter Dataset Format

## 2D Sweep Dataset

Each row represents one simulation.

```
mapvars = [
 [20,200,5],
 [20,200,10],
 [25,300,5]
]
```

Representation:

```
Simulation × Parameter
```

---

## 3D Sweep Dataset

Supports parameters with multiple elements.

```
mapvars = [
 [[25,300,5],[25,300,10]],
 [[20,200,5],[20,200,10]]
]
```

Representation:

```
Simulation × Parameter × Element
```

# Struct Generation

The system constructs a hierarchical Octave struct based on mapping paths.

Example paths:

```
Common.Thermal.Twater
Common.Control.Targets.Vout
```

Generated struct:

```octave
simStruct = struct(
 'ModelVars',
 struct(
   'Common',
   struct(
     'Thermal',
     struct('Twater',0),
     'Control',
     struct(
       'Targets',
       struct('Vout',0)
     )
   )
 )
);
```

# Generated Octave Simulation Structure

The generated script typically contains:

1. Simulation parameter initialization
2. Model variable assignment
3. Simulation loop
4. Scope data extraction
5. Result storage

Example simulation loop:

```octave
for sim = 1:num_simulations

    simStruct.ModelVars.Common.Thermal.Twater = data{sim}{1}(1);

    results = plecs('simulate', model, simStruct);

    scope_data = plecs('get', results, 'ScopeName');

end
```

# Example End-to-End Workflow

## Step 1 — Define Parameter Sweep

```python
mapvars = [
 [[25,300,5,180],[25,300,10,280]],
 [[20,200,10,200],[20,300,10,250]]
]
```

---

## Step 2 — Extract Mappings

```python
mappings = octave_sweep_mapping(
    "ScriptBody.py",
    mapvars
)
```

---

## Step 3 — Generate Octave Script

```python
octave_code = octave_sweep_script(
    mapvars,
    sweepnames,
    mappings,
    model_vars_dict,
    scopes_list
)
```

---

## Step 4 — Save Script

```python
with open("ScriptBody_20260312_213544.m","w") as f:
    f.write(octave_code)
```

---

## Step 5 — Inject into PLECS

```python
inject_octave_simple(
    plecs_file_path="ACfilterOBC.plecs",
    output_file_path="ACfilterOBC.plecs",
    m_file_path="ScriptBody_20260312_213544.m"
)
```

# Performance Considerations

| Factor                   | Impact                                 |
| ------------------------ | -------------------------------------- |
| Number of simulations    | Directly increases runtime             |
| Number of scopes         | Increases output size                  |
| Struct complexity        | Slightly increases initialization time |
| Parameter dimensionality | Affects indexing overhead              |

# Best Practices

### Keep mappings clean

Only include variable assignments inside the marker section.

### Avoid manual edits

Generated scripts should not be manually modified.

### Use descriptive parameter names

Example:

```
Water_Temperature
Input_Voltage
Output_Current
```
