# PLECS-Octave Integration for Parameter Sweep Simulations

## Feature Overview

This Python-based tool enables automated parameter sweep simulations by integrating Octave scripts into PLECS models. It bridges the gap between PLECS circuit simulation and Octave's numerical computing capabilities, allowing engineers and researchers to:

- **Automate multiple simulations** with varying parameters
- **Extract simulation results** systematically
- **Create hierarchical parameter structures** that mirror the PLECS model organization
- **Generate complete Octave scripts** from template-based code
- **Inject scripts directly into PLECS files** for seamless execution

The tool is particularly valuable for design optimization, sensitivity analysis, and validation studies where multiple parameter combinations need to be evaluated.

---

## How It Works (Detailed)

### 1. **Parameter Mapping Extraction** (`octave_sweep_mapping`)

The tool parses a Python script file (ScriptBody.py) that contains variable mappings in a specific format:

**Input Script Format:**
```python
#! -------------------------------------------------------------------------------------Don't change above this line---------------------------------------------------------------------------
mdlVars["Common"]["Water_Temperature"] = mapVars[X1][0]         # Direct parameter
mdlVars["Common"]["Derived_Param"] = mdlVars["Common"]["Water_Temperature"] * 2  # Derived parameter
#! -------------------------------------------------------------------------------------Don't change under this line---------------------------------------------------------------------------
```

**Processing Steps:**
- Identifies the section between special comment markers
- Extracts each line containing `mdlVars` assignments
- Distinguishes between **direct parameters** (using `mapVars`) and **derived parameters** (using other `mdlVars` references)
- Converts Python syntax to Octave syntax (e.g., `**` to `^` for exponentiation)
- Builds hierarchical paths like `simStruct.ModelVars.Common.Water_Temperature`

### 2. **Dimension Detection** (`detect_mapvars_dimensions`)

Analyzes the structure of `mapvars` to determine if it's:

- **2D structure**: Simple matrix of parameter values (rows = simulations, columns = parameters)
  ```
  [[p11, p12, p13],   # Simulation 1
   [p21, p22, p23]]   # Simulation 2
  ```

- **3D structure**: Nested structure allowing multiple elements per parameter
  ```
  [[[p111, p112], [p121]],    # Simulation 1, Parameter 1 has 2 values
   [[p211], [p221, p222]]]    # Simulation 1, Parameter 2 has 1 value
  ```

This detection determines how parameters are indexed in the generated Octave code.

### 3. **Octave Struct Generation** (`format_octave_struct_from_mappings`)

Creates a hierarchical Octave structure that mirrors the PLECS model organization:

```octave
simStruct = struct(
    'ModelVars', struct(
        'Common', struct(
            'Water_Temperature', 0,
            'Input_Voltage', 0,
            'Control', struct(
                'Targets', struct(
                    'Vout', 0,
                    'Pout', 0
                )
            )
        )
    ),
    'SolverOpts', struct()  # Solver options placeholder
);
```

### 4. **Script Template Rendering** (`octave_sweep_script`)

Uses Jinja2 templating to generate the complete Octave script:

**Template Structure (octave_sweep_template.m.j2):**
- Initialization of simulation parameters
- Loading of the PLECS model
- Loop structure for parameter sweeps
- Result storage and formatting
- Scope data extraction

**Key Features in Generated Script:**
- Automatic creation of results directory
- Progress tracking during simulation
- Comprehensive error handling
- Data organization by parameter combination

### 5. **PLECS File Injection** (`inject_octave_simple`)

Handles three scenarios when injecting the script into a PLECS file:

**Case 1 - Empty Script Section:**
- Finds `Script { Name "Script" Script "" }` pattern
- Replaces with populated script content

**Case 2 - Existing Script Section:**
- Appends new script after the last existing script
- Preserves multiple script sections

**Case 3 - No Script Section:**
- Inserts before final closing brace or appends at end
- Creates new script section from scratch

The injection process escapes special characters to maintain valid PLECS XML format.

## Example Usage (Detailed)

### Step 1: Define Parameter Sweep Space

```python
# 3D parameter structure with multiple values per parameter
mapvars = [
    # Simulation 1
    [
        [25, 30, 35],        # Water Temperature (3 values)
        [300, 310, 320],      # Input Voltage (3 values)
        [5, 6, 7],           # Output Current (3 values)
        [180, 190]           # Additional parameter (2 values)
    ],
    # Simulation 2
    [
        [20, 25],             # Water Temperature (2 values)
        [200, 210, 220],      # Input Voltage (3 values)
        [10],                 # Output Current (1 value)
        [200]                 # Additional parameter (1 value)
    ]
    # More simulations...
]

# Parameter names for reference
sweepnames = ["Water Temperature", "Input Voltage", "Output Current", "Additional Param"]
```

### Step 2: Create Mapping Script (ScriptBody.py)

```python
#! -------------------------------------------------------------------------------------Don't change above this line---------------------------------------------------------------------------
# Direct parameter assignments (connected to mapVars)
mdlVars["Common"]["Thermal"]["Twater"] = mapVars[X1][0]                 # First value of parameter 1
mdlVars["Common"]["Electrical"]["Vin"] = mapVars[X2][0]                 # First value of parameter 2
mdlVars["Common"]["Electrical"]["Iout"] = mapVars[X3][0]                # First value of parameter 3
mdlVars["Common"]["Electrical"]["Additional"] = mapVars[X4][0]          # First value of parameter 4

# Derived parameters (computed from other parameters)
mdlVars["Common"]["Control"]["Power_Limit"] = mdlVars["Common"]["Electrical"]["Vin"] * mdlVars["Common"]["Electrical"]["Iout"]
mdlVars["Common"]["Thermal"]["Temp_Rise"] = mdlVars["Common"]["Electrical"]["Iout"] * 2.5

# Array-type parameters (using additional indices)
mdlVars["Common"]["Profile"]["Temperature_Curve"] = [mapVars[X1][1], mapVars[X1][2]]  # Using multiple values
#! -------------------------------------------------------------------------------------Don't change under this line---------------------------------------------------------------------------
```

### Step 3: Generate Octave Script

```python
# Extract mappings from ScriptBody.py
mappings = octave_sweep_mapping(r'C:\Models\ScriptBody.py', mapvars)

# Define scopes to capture
scopes_list = ['Input_Voltage_Scope', 'Output_Current_Scope', 'Temperature_Scope']

# Generate the Octave script
octave_code = octave_sweep_script(
    mapvars, 
    sweepnames, 
    mappings, 
    model_vars_dict, 
    scopes_list
)

# Save with timestamp
timestamp = datetime.datetime.utcnow().strftime("%Y%m%d_%H%M%S")
with open(f'ScriptBody_{timestamp}.m', 'w') as f:
    f.write(octave_code)
```

### Step 4: Generated Octave Script (Excerpt)

```octave
% Auto-generated Octave script for parameter sweep
% Generated on: 2026-03-13 10:30:45

% Initialize results directory
results_folder = '/workspace/simulation_results';
mkdir(results_folder);

% Parameter definitions
mapvars = {...};  % Parameter values from Python

% Parameter names
sweepnames = {'Water Temperature', 'Input Voltage', 'Output Current'};

% Initialize simulation structure
simStruct = struct('ModelVars', struct(
    'Common', struct(
        'Thermal', struct('Twater', 0),
        'Electrical', struct('Vin', 0, 'Iout', 0),
        'Control', struct('Power_Limit', 0)
    )
));

% Main sweep loop
for sim = 1:size(mapvars, 1)
    fprintf('Running simulation %d of %d\n', sim, size(mapvars, 1));
    
    % Assign parameters for this simulation
    simStruct.ModelVars.Common.Thermal.Twater = data{sim}{1}(1);
    simStruct.ModelVars.Common.Electrical.Vin = data{sim}{2}(1);
    simStruct.ModelVars.Common.Electrical.Iout = data{sim}{3}(1);
    
    % Run simulation
    model = 'ACfilterOBC.plecs';
    results = plecs('simulate', model, simStruct);
    
    % Extract scope data
    for s = 1:length(scopes)
        scope_data = plecs('get', results, scopes{s});
        save(fullfile(results_folder, sprintf('sim%d_%s.mat', sim, scopes{s})), 'scope_data');
    end
end
```

### Step 5: Inject into PLECS File

```python
inject_octave_simple(
    plecs_file_path=r'C:\Models\ACfilterOBC.plecs',
    output_file_path=r'C:\Models\ACfilterOBC_with_sweep.plecs',
    m_file_path=r'C:\Models\ScriptBody_20260313_103045.m'
)
```

### Step 6: Execute in PLECS

1. Open the modified PLECS file in PLECS
2. Navigate to the Script section
3. Execute the script (or set to run automatically)
4. Results are saved to the `simulation_results` folder

### Expected Output Structure

```
simulation_results/
├── sim1_Input_Voltage_Scope.mat
├── sim1_Output_Current_Scope.mat
├── sim1_Temperature_Scope.mat
├── sim2_Input_Voltage_Scope.mat
├── sim2_Output_Current_Scope.mat
├── sim2_Temperature_Scope.mat
├── simulation_log.txt
└── parameter_summary.csv
```

Each `.mat` file contains the scope data for that simulation, ready for post-processing in Octave/MATLAB.
