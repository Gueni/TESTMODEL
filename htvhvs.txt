Looking at the `graph_scopes` function, the minimal change to support both 2D scatter and 3D plots when `permute = False` is to modify the data extraction logic for non-permuted (sequential) sweeps.

Currently, when `permute = False`, the code uses `zip_longest` which creates irregular combinations. Here's the minimal change needed:

## Change 1: Update the combination generation (around line ~658)

Replace this:
```python
all_combos = (list(dp.product(*sweep_vars.values())) if dp.JSON["permute"] else dp.np.where((arr := dp.np.array(list(dp.zip_longest(*sweep_vars.values(), fillvalue=None)), dtype=object)) == None, [v[-1] for v in sweep_vars.values()], arr).tolist())
```

With:
```python
if dp.JSON["permute"]:
    all_combos = list(dp.product(*sweep_vars.values()))
else:
    # For non-permuted sweeps, create sequential combinations
    # Pad shorter lists with their last value to match the longest list's length
    sweep_values_list = list(sweep_vars.values())
    max_len = max(len(v) for v in sweep_values_list)
    padded_combos = []
    for i in range(max_len):
        combo = tuple(v[i] if i < len(v) else v[-1] for v in sweep_values_list)
        padded_combos.append(combo)
    all_combos = padded_combos
```

## Why this works:

This change ensures that when `permute = False`:
1. **2D plots**: The x-axis values will be properly paired with y-values in sequential order (first with first, second with second, etc.)
2. **3D plots**: The surface mesh will be constructed from properly aligned (x,y,z) triplets instead of irregular combinations

The scatter plots (both 2D and 3D) will work correctly because the data points will maintain their sequential relationship rather than creating all possible permutations. This preserves the natural order of your sweep parameters.
